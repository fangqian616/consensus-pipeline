## 🎯 这是什么？

共识管线用**结构化多Agent辩论**替代单AI生成。不同AI"部门"从各自专业视角论证、挑战对方假设、收敛到高质量共识产出。

**一个框架，两条管线：**

| | 🔬 学术管线 | 🎬 创意管线 |
|---|---|---|
| **输入** | 研究主题 | 剧本/故事 |
| **流程** | 检索→审校→辩论→综述 | 8部门辩论→分镜表→视频提示词 |
| **输出** | 文献综述+论文元数据 | 九宫格分镜表+逐镜视频提示词 |
| **成熟度** | ✅ v0.7.1 — 生产可用 | 🔄 v3.0 — 持续迭代中 |
| **适合** | 研究者、学生、分析师 | 动画制作者、内容创作者 |

## 🚀 快速开始

```bash
git clone https://github.com/fangqian616/consensus-pipeline.git
cd consensus-pipeline
pip install -r requirements.txt
streamlit run app.py
```

### 学术管线（命令行）

```bash
# 设置DeepSeek API Key
export DEEPSEEK_API_KEY=your_key_here

# 运行完整管线
python run_pipeline.py --topic "机器学习在能源经济学上的运用"
```

### 学术管线（Streamlit界面）

1. 侧边栏配置API Key
2. 输入研究主题
3. AI生成领域配置→确认
4. 观看论文检索、过滤、辩论过程
5. 获取带置信度标注的文献综述

### 创意管线

1. 侧边栏选择"动画模式"
2. 输入剧本
3. AI自动配置8个部门
4. 实时观看部门辩论
5. 导出九宫格分镜表+视频提示词

## 🔬 学术管线详情

### 11个研究部门

| 部门 | 职责 |
|------|------|
| 文献检索 | 多源检索（OpenAlex、Semantic Scholar、arXiv），去重 |
| 元数据检查 | DOI验证、元数据完整性检查 |
| 引用网络 | 引用分析、影响因子指标 |
| 方法论评审 | 7维度评估（准确性、效率、可解释性…） |
| 数据验证 | 数据源质量、可复现性评估 |
| 反证部门 | 反主流发现、争议映射 |
| 主题聚类 | 主题分组、趋势识别 |
| 可视化 | 图表与分布分析 |
| 报告整合 | 交叉验证、共识构建 |
| **编程** | 分析主流模型/工具，输出可运行代码 |
| **教程** | 教授研究工具与方法使用 |

### QC审校部门（v0.7.1新增）

v0.7.1最大的质量飞跃。三层过滤确保零污染：

```
输入：219篇论文（多源检索）
  │
  ├── 第1层：硬过滤（排除信号）
  │   → 直接踢出明显不相关论文（如碳纳米管、植物学）
  │   → 排除：5篇
  │
  ├── 第2层：LLM分类（领域归属）
  │   → LLM逐篇判定："是否属于本领域？"是/否
  │   → 排除：137篇
  │
  └── 第3层：层级标注（重要性分级）
      → core（68篇）/ method（6篇）/ background（3篇）
      → 最终产出：77篇

排除率：64.8%
最终产出不相关率：0%
```

### 动态领域配置（v0.7.1新增）

不再硬编码关键词。LLM根据你的主题生成一切：

```json
{
  "domain_definition": "ML方法在能源经济学中的应用...",
  "exclusion_signals": ["碳纳米管", "植物学", "材料科学"],
  "query_rotation": ["机器学习 能源预测", "深度学习 电价", ...],
  "tier_definitions": {"core": "...", "method": "...", "background": "..."},
  "llm_classify_prompt": "给定论文标题和摘要..."
}
```

主题从"能源经济学"换成"医疗LLM"？零代码改动。

### 置信度标注

报告中每条结论都带置信度标签：

> 深度学习方法主导短期能源负荷预测 **（42/77篇支撑，高置信度）**

> 图神经网络在能源网络优化中显示新兴潜力 **（3/77篇支撑，低置信度——趋势未建立）**

不再有无支撑的断言。

## 🎬 创意管线详情

### 8个创意部门

| 部门 | 辩手 | 侧重 |
|------|------|------|
| 编剧 | 微表情/肢体语言/情绪节奏/叙事架构师 | 角色细节、叙事节拍、出场清单 |
| 空间 | 场景测量/走位调度/空间逻辑 | 布局、站位、移动路径 |
| 分镜 | 长镜头/蒙太奇/冲击帧 | 镜头构成、九宫格关键帧 |
| 摄影 | 镜头写实/光影视角/运动设计 | 焦段、景深、运镜 |
| 灯光 | 自然光/戏剧光/实用光 | 光源、氛围、对比 |
| 特效 | 粒子/物理/合成 | 效果、破坏、能量 |
| 音效 | 环境音/拟音/配乐占位 | 环境声、动作音 |
| 剪辑 | 叙事节奏/视觉连贯/镜头完整 | 切点、分段 |

### 输出格式（v3.1实测验证）

每个镜头遵循实测验证的模板：

```
SHOT 01 | 场景名 | 0-2s
参考分镜图：九宫格格子位置
运镜：...
打光：...
引导：...
主体：角色 [角色 参考图] + 动作描述
背景虚化：...

正面提示词：（英文，完整运镜+画面+光影+渲染描述）
音频：（仅音效，绝对不含BGM）
```

迭代踩出的关键规则：
- **一镜头一动作**——单SHOT内不做动作弧线
- **绝对方向**——"面朝车厢后方（沙虫来向）"，不用"面朝镜头"
- **参考图标识**——`角色 [角色 参考图]` 代替外貌描述
- **车内车外分离**——A线（车内）和B线（车外）独立文件

## ✨ 核心特性

### 🧠 AI Router — 智能配组
AI分析你的需求→选择部门→配置辩手→生成预设。全部可编辑。

### 🔄 四种架构模式

| 模式 | 机制 | 质量 | 速度 | 适合 |
|------|------|:---:|:---:|------|
| **共识管线** | 串行部门辩论 | ★★★★★ | ★★ | 最高质量 |
| **市场模拟** | 竞选+投票 | ★★★★ | ★★ | 探索方向 |
| **专家池** | 每部门精选2人 | ★★★ | ★★★ | 快速迭代 |
| **单Agent** | 无辩论 | ★★ | ★★★★★ | 对照基线 |

### 📦 技能注入
上传专属知识（Markdown）→自动注入目标部门提示词。

### 🔍 事实核查
报告主张的自动化事实验证，带来源追踪。

## 🔧 配置

| 提供商 | API地址 | 推荐模型 |
|--------|--------|---------|
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` 或 `deepseek-v4-pro` |
| OpenAI | `https://api.openai.com/v1` | `gpt-4o` |
| 自定义 | 任意OpenAI兼容端点 | 任意模型 |

### 可选：easyScholar API

增强期刊排名，在 [easyScholar](https://www.easyscholar.cc/) 获取密钥：

```bash
# 在Streamlit侧边栏或环境变量中设置
export EASYSCHOLAR_SECRET_KEY=your_key_here
```

不使用easyScholar时，管线使用本地209期刊注册表作为回退。

### 环境变量

```bash
DEEPSEEK_API_KEY=your_key          # LLM调用必需
EASYSCHOLAR_SECRET_KEY=your_key    # 可选，用于期刊排名
```
