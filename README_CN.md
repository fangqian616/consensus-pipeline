## ❓ 为什么不直接问ChatGPT？

单个LLM产出的答案听起来很自信，但没有交叉验证——幻觉混入其中，冲突视角被压平，你分不清哪些结论可靠、哪些是推测。本框架用**结构化多Agent辩论作为质量门控**替代一次性生成：每条主张都被独立"部门"挑战，矛盾被显式暴露，最终结论带有**置信度标注**（如"42/77篇支撑，高置信度"）。相当于自带同行评审——不是一个作者，而是一个对抗性委员会。

---

## 🎯 这是什么？

共识管线用**结构化多Agent辩论**替代单AI生成。不同AI"部门"从各自专业视角论证、挑战对方假设、收敛到高质量共识产出。

**核心设计理念：** 结构化多Agent辩论即质量门控——每条主张都被质证、每个假设都被挑战，产出带显式置信度而非虚假确定性。

**一个框架，两条管线：**

| | 🔬 学术管线 | 🎬 创意管线 |
|---|---|---|
| **输入** | 研究主题 | 剧本/故事 |
| **流程** | 检索→审校→辩论→综述 | 8部门辩论→分镜表→视频提示词 |
| **输出** | 文献综述+论文元数据 | 九宫格分镜表+逐镜视频提示词 |
| **成熟度** | 🧪 v0.7.3 — 早期阶段，寻求反馈 | 🔄 v3.0 — 持续迭代中 |
| **适合** | 研究者、学生、分析师 | 动画制作者、内容创作者 |

## 🔬 学术管线详情

### 11个研究部门

| 部门 | 职责 |
|------|------|
| 文献检索 | 多源检索（OpenAlex、Semantic Scholar、arXiv），去重 |
| 元数据检查 | DOI验证、元数据完整性检查 |
| 引用网络 | 引用分析、影响因子指标 |
| 方法论评审 | 7维度评估（准确性、效率、可解释性…） |
| 数据验证 | 数据源质量、可复现性评估 |
| 反证部门 | 反主流发现、争议映射 |
| 主题聚类 | 主题分组、趋势识别 |
| 可视化 | 图表与分布分析 |
| 报告整合 | 交叉验证、共识构建 |
| **编程** | 分析主流模型/工具，输出可运行代码 |
| **教程** | 教授研究工具与方法使用 |

### QC审校部门（v0.7.1+）

v0.7.1最大的质量飞跃。三层过滤确保零污染：

- **第1层——硬过滤**：通过排除信号直接踢出明显不相关论文（如碳纳米管、植物学）
- **第2层——LLM分类**：LLM逐篇判定领域归属；排除137/219篇
- **第3层——层级标注**：将剩余论文按重要性分级：core（68篇）/ method（6篇）/ background（3篇）

结果：219→77篇，排除率64.8%，最终产出不相关率0%。

### 动态领域配置（v0.7.1+）

不再硬编码关键词。LLM根据你的主题生成一切：

```json
{
  "domain_definition": "ML方法在能源经济学中的应用...",
  "exclusion_signals": ["碳纳米管", "植物学", "材料科学"],
  "query_rotation": ["机器学习 能源预测", "深度学习 电价", ...],
  "tier_definitions": {"core": "...", "method": "...", "background": "..."}
}
```

主题从"能源经济学"换成"医疗LLM"？零代码改动。

### 置信度标注

报告中每条结论都带置信度标签：

> 深度学习方法主导短期能源负荷预测 **（42/77篇支撑，高置信度）**

> 图神经网络在能源网络优化中显示新兴潜力 **（3/77篇支撑，低置信度——趋势未建立）**

不再有无支撑的断言。

## 🎬 创意管线详情

### 8个创意部门

| 部门 | 辩手 | 侧重 |
|------|------|------|
| 编剧 | 微表情/肢体语言/情绪节奏/叙事架构师 | 角色细节、叙事节拍、出场清单 |
| 空间 | 场景测量/走位调度/空间逻辑 | 布局、站位、移动路径 |
| 分镜 | 长镜头/蒙太奇/冲击帧 | 镜头构成、九宫格关键帧 |
| 摄影 | 镜头写实/光影视角/运动设计 | 焦段、景深、运镜 |
| 灯光 | 自然光/戏剧光/实用光 | 光源、氛围、对比 |
| 特效 | 粒子/物理/合成 | 效果、破坏、能量 |
| 音效 | 环境音/拟音/配乐占位 | 环境声、动作音 |
| 剪辑 | 叙事节奏/视觉连贯/镜头完整 | 切点、分段 |

### 输出格式（v3.1实测验证）

每个镜头遵循实测验证的模板：

```
SHOT 01 | 场景名 | 0-2s
参考分镜图：九宫格格子位置
运镜：...
打光：...
引导：...
主体：角色 [角色 参考图] + 动作描述
背景虚化：...
正面提示词：（英文，完整运镜+画面+光影+渲染描述）
音频：（仅音效，绝对不含BGM）
```

迭代踩出的关键规则：
- **一镜头一动作**——单SHOT内不做动作弧线
- **绝对方向**——"面朝车厢后方（沙虫来向）"，不用"面朝镜头"
- **参考图标识**——`角色 [角色 参考图]` 代替外貌描述
- **车内车外分离**——A线（车内）和B线（车外）独立文件

## ❓ 常见问题 / 故障排除

**Q: 完整管线运行需要多长时间？**
A: 学术管线：10-30分钟，取决于主题和论文数量。创意管线：8部门辩论5-15分钟。使用"专家池"模式可缩短约50%。

**Q: Token消耗大吗？费用多少？**
A: 完整8部门创意辩论约10k-30k token（视辩论轮次而定）。专家池模式减少约50%。以DeepSeek定价，一次完整运行费用不到¥0.7。

**Q: 支持哪些API？**
A: 任何OpenAI兼容API。推荐DeepSeek（性价比最高），也支持OpenAI或本地部署模型。

**Q: 输出支持哪些语言？**
A: 学术管线：中文（`--lang zh`，默认）和英文（`--lang en`）。创意管线：提示词始终为英文；Streamlit界面支持中英文。

**Q: 辩论结果质量不稳定怎么办？**
A: (1) 增加辩论轮次 (2) 使用更强的模型 (3) 在提示词中增加更具体的约束 (4) 使用导演打回功能重辩。

**Q: 可以只跑部分部门吗？**
A: 可以。在智能配组中禁用不需要的部门即可。命令行模式运行全部部门。

**Q: 如何切换到不同的研究领域？**
A: 直接修改`--topic`参数。动态领域配置（v0.7.1+）会自动生成新的排除信号、搜索词轮换和分级定义——无需修改代码。

### 可选：easyScholar API

增强期刊排名，在 [easyScholar](https://www.easyscholar.cc/) 获取密钥：

```bash
# 在Streamlit侧边栏或环境变量中设置
export EASYSCHOLAR_SECRET_KEY=your_key_here
```

不使用easyScholar时，管线使用本地209期刊注册表作为回退。
