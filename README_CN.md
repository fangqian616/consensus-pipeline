## 🚀 快速开始

```bash
git clone https://github.com/fangqian616/consensus-pipeline.git
cd consensus-pipeline
pip install -r requirements.txt
streamlit run app.py
```

浏览器打开后，在侧边栏配置API密钥，然后：

1. **Tab 0** → 描述需求（AI访谈或直接填写）
2. **Tab 1** → AI自动配组部门和辩手
3. **Tab 2** → 填写研究主题/脚本
4. **Tab 3** → 观看各部门实时辩论
5. **Tab 4** → 跨部门辩论
6. **Tab 5** → 最终交付报告 + PDF导出

## 🔬 学术模式

v4.0的核心功能。完整的学术研究管线：

### 11个研究部门

| 部门 | 职责 |
|------|------|
| 文献检索 | 多源检索、去重 |
| 元数据审查 | DOI验证、元数据完整性 |
| 引用网络 | 引用分析、影响因子 |
| 方法论审查 | 7维度评估（精度、效率、可解释性…） |
| 数据验证 | 数据源质量、可复现性 |
| 反面证据 | 反主流发现、争议图谱 |
| 主题聚类 | 主题分组、趋势识别 |
| 可视化 | 图表与分布分析 |
| 报告整合 | 交叉验证、共识构建 |
| **程序部** | 分析主流模型/工具链，输出可运行代码 |
| **教程部** | 教学如何使用研究工具与方法 |

### 论文质量筛选（四道筛子）

```
第一道：easyScholar API → 期刊等级（S/A/B/C）
第二道：引用数 + h-index过滤
第三道：本地209期刊注册表（9学科 + 6个AI顶会 + 3个中文CSSCI）
第四道：相关性评分（_compute_relevance）——领域必须词 + ML必须词
```

**效果（ML在能源经济学实验）：**

| 指标 | v4.5（无easyScholar） | v5.0（easyScholar） | v5.1（全管线） |
|------|:---:|:---:|:---:|
| S级论文 | 3 | 47 | 41 |
| C级论文 | 80 | 9 | 9 |
| 论文总数 | 84 | 71 | 76 |
| 自我评估 | 4/10 | 4/10 | 5.5/10 |

### 双模板报告

- **交付报告**（≤3000字符）：核心发现+置信度、方法论分布、代表性论文、研究空白
- **内部文档**（完整详情）：全部辩论记录、论文元数据、程序与教程产出

均可导出PDF，支持中文字体（霞鹜文楷）。

## ✨ 核心特性

### 🧠 AI Router 智能配组
AI分析需求 → 选择部门 → 配置辩手 → 生成预设。全部可编辑。

### 💬 需求调研（v4.0+）
主管线前的AI访谈（Phase 0-4），通过对话定义范围、约束和优先级。

### 🏪 市场模拟模式
创意想法像产品一样竞争：候选方案 → 质量拷问 → 投票 → 补丁优化

### 🔄 四种架构模式

| 模式 | 机制 | 质量 | 速度 | 适合 |
|------|------|:---:|:---:|------|
| **共识管线** | 串行部门辩论 | ★★★★★ | ★★ | 最高质量 |
| **市场模拟** | 竞争+投票 | ★★★★ | ★★ | 探索方向 |
| **专家池** | 每部门2人精选 | ★★★ | ★★★ | 快速迭代 |
| **单智能体** | 无辩论 | ★★ | ★★★★★ | 基线对比 |

### 📦 Skill注入
上传专有知识（Markdown）→ 自动注入目标部门的辩手提示词。

### 🔍 事实校验（Phase 7.5）
自动验证报告中的事实性陈述，附带来源追踪。

## 🔧 配置

| 提供商 | API地址 | 推荐模型 |
|--------|--------|---------|
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| OpenAI | `https://api.openapi.com/v1` | `gpt-4o` |
| 自定义 | 任何兼容端点 | 任意模型 |

### 可选：easyScholar API

增强期刊分级，在 [easyScholar](https://www.easyscholar.cc/) 获取密钥：

```bash
# 在Streamlit侧边栏或环境变量中设置
export EASYSCHOLAR_SECRET_KEY=your_key_here
```

无easyScholar时，管线使用本地209期刊注册表作为回退。

## 📁 项目结构

```
consensus-pipeline/
├── app.py                    # Streamlit主应用（9个Tab）
├── router.py                 # AI Router——智能部门配置
├── debate_engine.py          # 核心辩论引擎
├── config_manager.py         # 配置持久化与预设
├── pdf_exporter.py           # PDF导出（中文字体）
├── requirement/              # 需求调研模块（v4.0+）
│   ├── interviewer.py        # AI访谈智能体
│   ├── structurer.py         # 范围与约束提取
│   ├── generator.py          # 需求文档生成
│   ├── validator.py          # 完整性检查
│   └── __init__.py
├── academic/                 # 学术研究模块（v4.0+）
│   ├── search_engine.py      # 多源检索（OpenAlex, Semantic Scholar, arXiv）
│   ├── journal_classifier.py # 四道期刊质量筛子 + easyScholar
│   ├── journal_registry.py   # 209期刊本地注册表
│   ├── report_generator.py   # 双模板报告生成
│   ├── fact_checker.py       # 自动事实校验
│   └── __init__.py
├── templates/                # 辩论提示词模板
├── presets/                  # 内置预设（动画、学术）
├── fonts/                    # 中文字体（霞鹜文楷）
└── run_pipeline.py           # CLI无头运行脚本
```
